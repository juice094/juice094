## 面试当天 Checklist

### 面试前 30 分钟
- [ ] 电脑充电、网络检查
- [ ] 打开项目 README（方便 share screen）
- [ ] 打开 Clarity/Devbase/Syncthing-rust 的 GitHub 页面
- [ ] 准备白板工具（Excalidraw / 飞书文档 / draw.io）
- [ ] 一杯水在手边

### 面试中
- [ ] 听到问题后停顿 2 秒再回答（不要抢答）
- [ ] 不确定时说"我可以从两个角度考虑这个问题..."
- [ ] 如果卡住："让我先整理一下思路"
- [ ] 代码题：先说思路 → 确认理解正确 → 再写代码
- [ ] 系统设计题：先澄清需求 → 再画架构图

### 面试后
- [ ] 记录被问的问题和你的回答质量
- [ ] 记录反问环节面试官透露的信息
- [ ] 准备下一轮

## 心理建设

### 核心信念

1. **你的 GitHub 是你真正的简历** — 1554 个通过的测试比学校排名更能证明能力
2. **Rust + AI Agent = 稀缺组合** — 同时懂 Rust 系统编程和 AI Agent 架构的应届生非常少
3. **面试是双向选择** — 你在评估团队是否适合你，不只是他们评估你
4. **每一次面试都是免费的成长机会** — 面试官的追问是最好的学习材料

### 面对质疑的准备

| 质疑 | 转化为优势 |
|------|-----------|
| "你是本科生" | "我用 3 个生产级项目证明了我的独立交付能力" |
| "没有实习经历" | "独立管理 22 crate workspace 的完整生命周期" |
| "学校不是985/211" | "开源项目不受学校限制，代码质量自己说话" |
| "没有论文" | "独立完成 RAG 系统性实证研究，数据驱动验证假设" |
